Standalone

With an appropriate file suffix and an association to a Swiby launcher, as zipped up application can run on the desktop by being double clicked, or in a web page just like Flash applications.

The Sweb

The web server hosts a Swiby application to a browser. The browser either natively or via plugin support swiby pages and renders them in a way that is very familiar to users of the the web.

Swiby pages -

  • are either static on the web server or part of a dynamic application ( think Ruby on Rails or Waffle making Swiby pages instead of HTML+JS )

  • have flow control - the eqivalent of <a href="nextpage.html">next page</a> will exist as a function on a 'context' object that is in scope. It implicitly causes the current page to be exited.

  • being text over the wire, are indexable by search engines (assuming an upgraded to be able to parse them).

  • like web pages, have a low startup and upgrade cost for end-users

  • unline web pages, are a single language (Ruby) rather than two (HTML + JavaScript)
